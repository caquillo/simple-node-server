Email

Today, Gmail and Yahoo are just some of the most common MUA - or Mail User Agents - being used to, well, send emails. When sending an electronic mail, the MUA sends the email to the Mail Delivery Agent (MDA); if the sender and receipient are from the same area, then they use the same MDA. But if the email has to be sent through a wider area network (WAN, in short), it has to go through the Internet cloud that consists of multiple mail servers, DNS, and routers. Some transmitters could get lost or take longer in the process because of the congestion in the Internet. To prevent such delays, e-mails have to be properly encapsulated with DNS, IP address, and sequence numbers so MDAs could send emails in the right direction. Having the correct DNS will allow the email to pass through firewalls and spam filters. To further prevent delays, MUAs often have their own mail servers that checks which MDA is available. It goes through the host and host chains, in which each MDA checks the IP address that is located on the header of the encapsulated data hence the next MDA could be identified. Most emails take the path that is dependent on server availability. Eventually the Mail Transfer Agent (MTA) will locate the local domain information then asks the appropriate DNS for proper information.
Another way of sending emails is through standard clients like Outlook. Unlike MUAs, emails are sent using SMTP (Simple Mail Transport Protocol) which goes like this: email is sent from the cloud to a server >> email goes through every email server until it finally lands on the receipient's server >> receipient either uses POP3 or IMAP, which are servers for receiving email. When (or rather if) the receiver decides to reply, he does so through SMTP once again. The flow of the email is the same as the flow of the initial email but with a few exeptions: POP3 is the only server being used this time to receive the email. Should the sender use a web page client, he makes use of HTTP, instead of SMTP, to connect to a web page on the Internet. Take note that the e-mail server needs to support the web mail. The sender then uses a browser such as Google Chrome or Safari to connect to the Internet and compose an email; transfer of email is done through SMTP. The receipient this time has three options to retrieve the email - either using POP3, IMAP4, or him accessing his email on the Internet using HTTP. In the last option, the email stays on the Internet server but is not transferred to the receipient.
